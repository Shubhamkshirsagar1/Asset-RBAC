# Phase 6: React Demo App — Implementation Plan

> REQUIRED SUB-SKILL: superpowers:executing-plans. Steps use checkbox syntax.

**Goal:** A lightweight Vite + React app in `web/` that demonstrates the backend: login with tenant
slug, a **dynamic nav rendered from `/me/menu`**, hospital (assets/work-orders) + PM
(projects/tasks) screens with **permission-gated buttons**, and an **admin console** with a visual
**grant/condition builder**, per-role **page toggles**, and the **"Why?" explainer** (`/admin/explain`).

**Architecture:** Vite dev server proxies backend routes to `localhost:3000` (no CORS changes, no
backend edits). A small `AuthContext` holds the JWT (localStorage), current user, effective
permissions, and menu. Client-side permission checks only **gray out** controls — the server stays
the source of truth. Plain CSS, `react-router-dom` for routing. No automated FE tests; verification
is `npm run build` + a dev-server smoke check.

**Tech Stack:** Vite 5, React 18, react-router-dom 6.

## Task 2: api + auth context

`api.js`:
```javascript
let token = localStorage.getItem('token') || null;
export const setToken = (t) => { token = t; t ? localStorage.setItem('token', t) : localStorage.removeItem('token'); };
export const getToken = () => token;
export async function api(path, { method = 'GET', body } = {}) {
  const res = await fetch(path, {
    method,
    headers: { 'content-type': 'application/json', ...(token ? { authorization: `Bearer ${token}` } : {}) },
    body: body ? JSON.stringify(body) : undefined,
  });
  const data = res.status === 204 ? null : await res.json().catch(() => null);
  if (!res.ok) throw Object.assign(new Error(data?.error || res.statusText), { status: res.status, data });
  return data;
}
```
`auth.jsx`: `AuthProvider` loads `/me`, `/me/permissions`, `/me/menu` after login; exposes
`{ user, permissions, menu, login(tenantSlug,email,password), logout, refresh }`.

## Task 5: domain screens (hospital + pm)

`Assets.jsx`: `GET /assets` table (name, status, value); "Create" (gated by `can(...,'asset','create')`),
"Dispose" per row (gated `asset:dispose`). `WorkOrders.jsx`: list; "Create"; per-row "Approve"
(gated `work_order:approve`) showing the server's 200/403 result inline. `Projects.jsx` /
`Tasks.jsx`: analogous (create project, create task, "Complete" gated `task:complete`). Buttons that
fail server-side surface the returned error (so a 403 is visible, proving server enforcement).

## Task 6: Admin console

`Admin.jsx` sections (only useful to `rbac:manage` users; non-admins see 403s gracefully):
- **Roles:** list roles (`GET /admin/roles`).
- **Grant builder:** pick role + resourceType + action + scope; optional **condition** via a small
  JSON textarea (with a couple of one-click presets: ownership, cost ≤ N, status ≠ done); POST to
  `/admin/roles/:id/grants`.
- **Page toggles:** list pages; toggle a page on/off for a role (`PUT /admin/roles/:r/pages/:p`).
- **Why? explainer:** form (userId, action, resourceType, JSON resource) → `POST /admin/explain` →
  render `{ allowed, scope, reason, matchedGrant }`.

- [ ] Build.
